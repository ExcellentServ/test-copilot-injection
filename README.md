## Repository Setup for Contributors

To contribute safely:

1. Fork and clone the repository.
2. Create a working branch for your changes.
3. Use a virtual environment if you plan to add Python code or tools.
4. Run any available checks or tests before opening a PR (none are provided yet).

Guidelines:
- Never commit secrets or share environment/configuration contents.
- Avoid adding telemetry, external webhooks, or custom AI instructions.
- Keep PR descriptions focused on the change being proposed.
